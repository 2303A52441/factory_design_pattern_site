Factory Design Pattern

A simple JavaScript example showing how a factory creates objects without exposing the creation logic.

What is the Factory Pattern?

The Factory Pattern provides an interface for creating objects in a superclass, but allows subclasses or a specialized ‚Äúfactory‚Äù object to decide which class to instantiate.

Instead of calling new directly everywhere, you ask a factory to give you an object.

Factory Pattern Code (JavaScript)

// Product classes
class Circle {
  constructor() {
    this.type = "Circle";
  }
  draw() {
    return "Drawing a circle ‚≠ï";
  }
}

class Square {
  constructor() {
    this.type = "Square";
  }
  draw() {
    return "Drawing a square ‚¨ú";
  }
}

class Triangle {
  constructor() {
    this.type = "Triangle";
  }
  draw() {
    return "Drawing a triangle üî∫";
  }
}

// Factory
class ShapeFactory {
  static createShape(shapeType) {
    switch (shapeType) {
      case "circle":
        return new Circle();
      case "square":
        return new Square();
      case "triangle":
        return new Triangle();
      default:
        throw new Error("Unknown shape type");
    }
  }
}

Why Use the Factory Pattern?

  • Centralizes object creation logic in one place.
  • Makes your code easier to extend (add new types later).
  • Reduces duplication of new SomeClass() everywhere.
